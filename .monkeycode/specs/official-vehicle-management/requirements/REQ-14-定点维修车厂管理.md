# REQ-14: 定点维修车厂管理 (V1)

**优先级**: P0
**版本**: V1

## 描述

定点维修车厂管理模块旨在为企业车队建立规范化的定点维修服务体系。覆盖维修厂从信息登记、资质审核、合作状态流转、服务能力配置、操作员管理到服务数据评估的全生命周期管理。

## 需求条目

### 第一节：维修厂信息登记

REQ-14-1-1: When 车队管理员新增维修厂时，the system shall 要求提供维修厂名称、统一社会信用代码、经营地址、联系人姓名、联系电话作为必填字段。

REQ-14-1-2: When 录入的统一社会信用代码与已有维修厂重复时，the system shall 拒绝保存并提示"该信用代码已存在"。

REQ-14-1-3: The system shall 支持录入经营范围描述作为选填字段。

REQ-14-1-4: The system shall 支持上传营业执照和维修资质证书，格式为JPG、PNG或PDF，单文件不超过10MB。

### 第二节：维修厂列表与查询

REQ-14-2-1: The system shall 提供维修厂列表页面，默认按名称升序排列，每页显示20条记录。

REQ-14-2-2: When 用户输入维修厂名称、地址或联系人关键词时，the system shall 支持模糊匹配查询。

REQ-14-2-3: The system shall 支持按定点状态（合作中、暂停合作、终止合作）筛选维修厂列表。

REQ-14-2-4: The system shall 支持按累计工单数、平均评分、合作时长进行排序。

REQ-14-2-5: When 车队管理员编辑维修厂信息时，the system shall 保留历史修改记录并记录修改人和修改时间。

### 第三节：定点合作关系管理

REQ-14-3-1: The system shall 维护定点状态为以下三种：合作中、暂停合作、终止合作。

REQ-14-3-2: When 车队管理员将维修厂设为合作中状态时，the system shall 要求录入合同编号、签约日期和合同有效期。

REQ-14-3-3: While 维修厂处于合作中状态时，the system shall 允许向该维修厂分配维修工单。

REQ-14-3-4: When 车队管理员将维修厂状态变更为暂停合作时，the system shall 阻止新的维修工单分配给该维修厂，已有工单不受影响。

REQ-14-3-5: When 车队管理员将维修厂状态变更为终止合作时，the system shall 将该维修厂从可选维修厂列表中移除，并自动禁用该维修厂下所有操作员账号。

REQ-14-3-6: If 维修厂合同到期日期前30天，then the system shall 生成合同到期提醒通知并推送至车队管理员。

REQ-14-3-7: When 合同到期时，the system shall 自动将维修厂状态变更为终止合作。

### 第四节：服务能力配置

REQ-14-4-1: When 车队管理员配置服务能力时，the system shall 支持勾选维修厂可提供的服务类型，包括机电维修、钣金喷漆、轮胎服务、美容养护、四轮定位、空调维修、电路检修、变速箱维修。

REQ-14-4-2: When 车队管理员设置营业时间时，the system shall 记录每日营业开始时间和结束时间。

REQ-14-4-3: The system shall 支持录入工位数量、拖车服务是否可用、是否提供代步车作为备选信息。

REQ-14-4-4: When 创建维修工单时，the system shall 在可选维修厂列表中仅展示处于合作中状态且服务类型匹配工单需求的维修厂。

### 第五节：维修厂人员账号管理

REQ-14-5-1: When 车队管理员为维修厂创建操作员账号时，the system shall 绑定该账号至指定维修厂并设置初始密码。

REQ-14-5-2: While 维修厂人员使用操作员账号登录时，the system shall 限定该账号仅能查看和处理本维修厂的工单。

REQ-14-5-3: If 维修厂合作关系终止，then the system shall 自动禁用该维修厂下所有操作员账号。

REQ-14-5-4: When 车队管理员重置维修厂人员密码时，the system shall 生成临时密码并强制首次登录修改。

### 第六节：维修厂服务看板

REQ-14-6-1: When 车队管理员查看维修厂服务看板时，the system shall 展示该维修厂的累计工单数、累计维修费用、平均维修周期和当前待处理工单数。

REQ-14-6-2: When 车队管理员查看维修厂服务看板时，the system shall 展示该维修厂的评价综合评分和评分趋势图。

REQ-14-6-3: When 车队管理员查看维修厂服务看板时，the system shall 展示最近12个月的月度工单量柱状图。

REQ-14-6-4: When 维修工单完成后，the system shall 支持车队管理员对维修厂进行多维度评价，维度包括维修质量、交付时效、服务态度、价格合理性。

REQ-14-6-5: The system shall 基于最近12个月的评价数据动态计算维修厂综合评级分数。

## 业务约束

1. 每个统一社会信用代码在系统中对应唯一一条维修厂记录
2. 维修工单分配的目标维修厂须满足 status=active 且 service_type 匹配工单需求
3. 每次状态变更须记录操作人、操作时间和变更前后状态
4. 维修厂操作员账号仅能操作本维修厂的数据，禁止跨厂访问
