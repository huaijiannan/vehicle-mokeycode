# REQ-10: 权限与角色管理 (V1)

**优先级**: P0
**版本**: V1（第一版基础功能）

## 描述

建立用户账号体系，通过角色控制不同用户的功能访问权限。

## 需求条目

### 第一节：角色定义

REQ-10-1-1: The system shall 提供以下预置角色：管理员、普通员工、部门负责人、分管领导、调度员、驾驶员、纪检监察、财务。

REQ-10-1-2: The system shall 为每个角色配置固定的菜单访问权限和功能操作权限。

### 第二节：用户管理

REQ-10-2-1: When 管理员创建用户时，the system shall 要求填写用户名、密码、真实姓名、角色、所属部门。

REQ-10-2-2: The system shall 支持启用/停用用户账号，停用账号无法登录。

REQ-10-2-3: The system shall 支持修改用户密码（管理员重置或用户自行修改）。

REQ-10-2-4: The system shall 支持更新用户所属部门和角色。

### 第三节：登录认证

REQ-10-3-1: When 用户登录时，the system shall 校验用户名和密码，签发有效期8小时的JWT token。

REQ-10-3-2: When token过期时，the system shall 返回401状态码要求重新登录。

REQ-10-3-3: The system shall 在登录成功后将用户信息和角色写入前端会话存储。

### 第四节：权限控制

REQ-10-4-1: The system shall 在后端API层进行角色权限校验，越权访问返回403状态码。

REQ-10-4-2: The system shall 在前端根据用户角色动态显示/隐藏菜单项。

## 关联接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 用户登录 |
| GET | `/api/auth/me` | 获取当前用户信息 |
| GET | `/api/users` | 用户列表 |
| POST | `/api/users` | 新增用户 |
| PUT | `/api/users/:id` | 更新用户 |
| PUT | `/api/users/:id/password` | 修改密码 |
| PUT | `/api/users/:id/status` | 启用/停用 |

## V2 预留

- 组织架构管理（多层级部门树、分管条线配置）
- 自定义角色与权限模板
- 权限申请审批流程
- 权限委托/代理（指定时间段代理他人权限）
- 三员分立（系统管理员/安全保密员/安全审计员）
- 操作日志与安全审计
- IP 访问白名单
