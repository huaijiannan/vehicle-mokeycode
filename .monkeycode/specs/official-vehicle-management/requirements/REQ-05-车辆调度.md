# REQ-05: 车辆调度 (V1)

**优先级**: P0
**版本**: V1（第一版基础功能）

## 描述

调度员为审批通过的用车申请分配车辆和驾驶员，生成派车单。

## 需求条目

### 第一节：待调度列表

REQ-05-1-1: When 审批通过后，the system shall 自动将该申请加入待调度列表。

REQ-05-1-2: The system shall 在待调度列表展示申请编号、申请人、用车事由、出发时间、乘车人数等关键信息。

REQ-05-1-3: The system shall 按出发时间排序，紧急申请标记优先展示。

### 第二节：派车操作

REQ-05-2-1: When 调度员选择车辆时，the system shall 列出状态为"空闲"且座位数满足乘车人数的车辆。

REQ-05-2-2: When 调度员选择驾驶员时，the system shall 列出状态为"空闲"且准驾车型匹配的驾驶员。

REQ-05-2-3: When 调度员确认派车时，the system shall 生成派车单，将车辆状态设为"出车中"，驾驶员状态设为"出车中"。

REQ-05-2-4: The system shall 向驾驶员推送出车任务通知，包含用车时间、目的地和车辆信息。

### 第三节：派车单管理

REQ-05-3-1: The system shall 支持查看所有派车记录，按日期筛选。

REQ-05-3-2: The system shall 在派车单详情展示关联的申请单信息、车辆信息和驾驶员信息。

REQ-05-3-3: When 某个车辆或驾驶员在申请时段已有派车记录时，the system shall 在调度界面标记为"已占用"。

## 状态机

```mermaid
stateDiagram-v2
    [*] --> 待调度 : 审批通过
    待调度 --> 已派车 : 调度确认
    已派车 --> [*]
```

## 关联接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/dispatch/pending` | 待调度列表 |
| POST | `/api/dispatch/assign` | 派车（指定车辆和驾驶员） |
| GET | `/api/dispatch` | 派车记录列表 |

## V2 预留

- 资源匹配引擎（根据场景自动推荐最佳车辆和驾驶员）
- 甘特图时间线可视化
- 紧急插队调度（高优先级申请优先分配）
- 调度冲突检测与智能解决
- 调度执行统计（日报/效率分析）
- 批量派车（一次性处理多个申请）
