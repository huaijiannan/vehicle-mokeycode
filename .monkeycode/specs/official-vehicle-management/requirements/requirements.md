# Requirements Document

## Introduction

定点维修车厂管理模块是车辆运营管理系统（vehicle-mokeycode）的核心子模块，旨在为企业车队建立规范化的定点维修服务体系。该模块覆盖维修厂从信息登记、资质审核、合作状态流转、服务能力配置、操作员管理到服务数据评估的全生命周期管理。

## Glossary

| 术语 | 定义 |
|------|------|
| 定点维修厂 | 与企业签订合作协议、由管理员在系统内登记的可承接车辆维修保养服务的维修机构 |
| 定点状态 | 维修厂与企业合作关系的状态标识，包括合作中、暂停合作、终止合作 |
| 合作中 | 维修厂正常运营状态，可接收维修工单分配 |
| 暂停合作 | 维修厂临时不可用状态，阻止新工单分配，已有工单不受影响 |
| 终止合作 | 维修厂合作关系结束，从可选列表中移除，操作员账号自动禁用 |
| 服务类型 | 维修厂可提供的维修服务大类，包括机电维修、钣金喷漆、轮胎服务、美容养护、四轮定位、空调维修、电路检修、变速箱维修 |
| 操作员 | 绑定至维修厂的系统登录账号，由车队管理员创建和管理 |
| 信用代码 | 统一社会信用代码，维修厂的唯一法定标识，18位字符 |
| 服务看板 | 维修厂的运营数据仪表盘，展示工单统计、费用汇总、评分趋势等关键指标 |

## 用户角色

- **车队管理员**：核心操作用户，负责维修厂的登记、编辑、状态管理、服务能力配置、操作员管理和数据查看
- **维修厂人员（操作员）**：由车队管理员创建的受限账号，仅可查看和处理本维修厂的工单
- **系统管理员**：负责维修厂操作员账号的密码重置等系统级操作

## Requirements

### R1 - 维修厂信息管理

**User Story:** AS 车队管理员, I want 登记和维护维修厂档案信息, so that 建立完整的维修厂信息库

#### Acceptance Criteria

1. WHEN 车队管理员新增维修厂，系统 SHALL 记录维修厂名称、统一社会信用代码、经营地址、联系人姓名、联系电话、经营范围描述
2. WHEN 车队管理员编辑维修厂信息，系统 SHALL 保留历史修改记录并记录修改人和修改时间
3. WHEN 车队管理员查看维修厂列表，系统 SHALL 支持按名称、地址、定点状态进行筛选和搜索
4. WHEN 车队管理员上传维修厂资质文件，系统 SHALL 支持营业执照、维修资质证书的图片或PDF上传存储
5. IF 录入的统一社会信用代码与已有维修厂重复，系统 SHALL 提示重复并拒绝保存

---

### R2 - 定点合作关系管理

**User Story:** AS 车队管理员, I want 管理维修厂定点合作状态和合同, so that 清晰掌握各维修厂的合作有效期和状态

#### Acceptance Criteria

1. WHEN 车队管理员将维修厂设为合作中状态，系统 SHALL 要求录入合同编号、签约日期和合同有效期
2. WHILE 维修厂处于合作中状态，系统 SHALL 允许向该维修厂分配维修工单
3. WHEN 车队管理员变更维修厂状态为暂停合作，系统 SHALL 阻止新的维修工单分配给该维修厂
4. WHEN 车队管理员变更维修厂状态为终止合作，系统 SHALL 将该维修厂从可选维修厂列表中移除
5. IF 维修厂合同到期日期前30天，系统 SHALL 生成合同到期提醒通知车队管理员

---

### R3 - 维修厂服务能力配置

**User Story:** AS 车队管理员, I want 配置维修厂的服务能力范围, so that 根据维修需求精准匹配维修资源

#### Acceptance Criteria

1. WHEN 车队管理员配置服务能力，系统 SHALL 支持勾选维修厂可提供的服务类型，包括机电维修、钣金喷漆、轮胎服务、美容养护、四轮定位、空调维修、电路检修、变速箱维修
2. WHEN 车队管理员为维修厂设置营业时间，系统 SHALL 记录每日营业开始时间和结束时间
3. WHEN 车队管理员设置维修厂备选信息，系统 SHALL 支持录入工位数量、拖车服务是否可用、是否提供代步车
4. WHEN 创建维修工单时，系统 SHALL 仅展示处于合作中状态且服务类型匹配的维修厂

---

### R4 - 维修厂人员账号管理

**User Story:** AS 车队管理员, I want 为维修厂创建操作员账号, so that 维修厂人员可以登录系统处理工单

#### Acceptance Criteria

1. WHEN 车队管理员为维修厂创建操作员账号，系统 SHALL 绑定该账号至指定维修厂并设置初始密码
2. WHEN 维修厂人员使用账号登录，维修厂人员 SHALL 仅能查看和处理本维修厂的工单
3. IF 维修厂终止合作，系统 SHALL 自动禁用该维修厂下所有操作员账号
4. WHEN 车队管理员重置维修厂人员密码，系统 SHALL 生成临时密码并强制首次登录修改

---

### R5 - 维修厂服务看板

**User Story:** AS 车队管理员, I want 查看每个定点维修厂的服务数据看板, so that 全面评估维修厂的服务质量和合作价值

#### Acceptance Criteria

1. WHEN 车队管理员查看维修厂看板，系统 SHALL 展示该维修厂的累计工单数、累计维修费用、平均维修周期、当前待处理工单数
2. WHEN 车队管理员查看维修厂看板，系统 SHALL 展示该维修厂的评价综合评分和评分趋势图
3. WHEN 车队管理员查看维修厂看板，系统 SHALL 展示最近12个月的月度工单量柱状图
4. WHEN 车队管理员查看维修厂列表，系统 SHALL 支持按累计工单数、平均评分、合作时长进行排序

## 业务约束

1. **信用代码唯一性**: 系统中每个统一社会信用代码对应唯一一条维修厂记录
2. **工单分配约束**: 维修工单分配的目标维修厂必须满足 status=active 且 service_type 匹配工单需求
3. **合同到期联动**: 合同到期次日自动将维修厂状态变更为 terminated，同时禁用该维修厂所有操作员账号
4. **操作员权限隔离**: 维修厂操作员账号仅能查看和处理本维修厂的工单，不能跨厂操作
5. **状态变更记录**: 每次状态变更记录操作人、操作时间、变更前后状态
