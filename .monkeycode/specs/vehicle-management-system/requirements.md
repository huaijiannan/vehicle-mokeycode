# Requirements Document

## Introduction

车辆管理系统（vehicle-mokeycode）是面向企业车队的智能化运营管理平台，覆盖加油卡管理、维修保养、保险理赔、年检管理、驾驶员管理、成本分析及移动端运营等核心业务场景。

## Glossary

| 术语 | 定义 |
|------|------|
| 主卡 | 企业统一持有的加油主账户卡片，可绑定多张副卡 |
| 副卡 | 分配给具体车辆或驾驶员的加油子卡片，由主卡统一管理额度和充值 |
| 维修厂 | 提供车辆维修保养服务的第三方合作机构 |
| 保险理赔 | 车辆发生事故后向保险公司申请赔付的完整流程 |
| 年检 | 车辆按照规定周期进行的年度安全技术检验 |
| 驾驶员扣分 | 驾驶员因交通违法行为被记分的记录 |
| 油耗异常 | 车辆油耗超出正常波动范围的异常状态 |
| TCO | Total Cost of Ownership，车辆全生命周期运营成本 |
| 同比 | 与去年同期相比 |
| 环比 | 与上一周期相比 |
| 组织架构 | 总公司-分公司-车队三级树形组织体系，上级可查看下级全部数据 |
| 通知渠道 | 仅包含站内消息和移动端推送，不涉及短信或邮件 |
| 车队管理员 | 系统核心用户，负责车辆、驾驶员、费用的全面管理 |
| 驾驶员 | 车辆的实际驾驶人员 |
| 财务人员 | 负责费用审核、报销审批和成本分析 |
| 维修厂人员 | 合作维修厂的操作人员，负责接单和维修记录 |
| 保险专员 | 负责保险理赔流程跟进的人员 |

## 用户角色

- 车队管理员
- 驾驶员
- 财务人员
- 维修厂人员
- 保险专员
- 系统管理员

## Requirements

### R1 - 加油卡统一管理

**User Story:** AS 车队管理员, I want 统一管理企业所有加油卡, so that 实时掌控油费支出、防止资金滥用

#### Acceptance Criteria

1. WHEN 车队管理员查看加油卡列表，系统 SHALL 以主副卡树形结构展示所有加油卡及其余额
2. WHEN 车队管理员为主卡充值，系统 SHALL 记录充值金额、充值时间、操作人，并自动更新主卡余额
3. WHEN 车队管理员向某张副卡分配额度，系统 SHALL 从主卡余额中扣减并增加副卡余额，同时生成分配记录
4. WHEN 任一加油卡余额低于预设阈值，系统 SHALL 自动生成余额不足预警通知
5. IF 副卡消费金额超过当日限额，系统 SHALL 拒绝交易并通知车队管理员

---

### R2 - 维修厂评价与评级体系

**User Story:** AS 车队管理员, I want 对合作维修厂进行评价和评级, so that 筛选优质维修资源、控制维修质量

#### Acceptance Criteria

1. WHEN 维修工单完成后，车队管理员 SHALL 对维修厂进行多维度评价，维度包括维修质量、交付时效、服务态度、价格合理性
2. WHEN 车队管理员查看维修厂列表，系统 SHALL 展示每家维修厂的综合评分和星级
3. WHILE 维修厂评级处于活跃状态，系统 SHALL 基于最近12个月的评价数据动态计算评级分数
4. IF 维修厂评级低于最低合格线，系统 SHALL 标记该维修厂为待观察状态并通知车队管理员
5. WHEN 驾驶员在维修厂完成服务后，驾驶员 SHALL 对该维修厂进行基础评分

---

### R3 - 保险理赔闭环管理

**User Story:** AS 保险专员, I want 完整的保险理赔流程管理, so that 加速理赔处理、减少资金占用

#### Acceptance Criteria

1. WHEN 车辆发生事故，保险专员 SHALL 创建理赔申请单，记录事故时间、地点、类型、损失描述及现场照片
2. WHEN 理赔申请提交，系统 SHALL 将理赔单状态标记为已报案并记录报案号
3. WHEN 定损完成后，保险专员 SHALL 录入定损金额和定损明细
4. WHEN 理赔款到账，保险专员 SHALL 更新理赔状态为已结案并记录到账金额
5. IF 理赔超过预设工作日未更新状态，系统 SHALL 生成超时预警并通知保险专员

---

### R4 - 年检到期自动提醒与预约管理

**User Story:** AS 车队管理员, I want 年检到期自动提醒和预约管理, so that 避免车辆脱检导致违规

#### Acceptance Criteria

1. WHEN 车辆年检到期日期前30天，系统 SHALL 自动生成年检提醒通知
2. WHEN 年检到期日期前14天且未完成预约，系统 SHALL 再次发送提醒
3. WHEN 车队管理员进行年检预约，系统 SHALL 记录预约检测站、预约时间和检测结果
4. IF 年检到期日期前7天仍未完成预约，系统 SHALL 升级提醒至红色预警并通知车队管理员上级
5. WHEN 年检完成后，车队管理员 SHALL 上传检测报告并更新下次年检到期日期

---

### R5 - 驾驶员扣分自动绑定

**User Story:** AS 车队管理员, I want 驾驶员交通违法扣分自动记录, so that 及时掌握驾驶员驾驶行为、降低安全风险

#### Acceptance Criteria

1. WHEN 车队管理员录入交通违法记录，系统 SHALL 自动将扣分绑定至对应驾驶员
2. WHILE 驾驶员存在未处理的交通违法记录，系统 SHALL 在驾驶员档案中展示累计扣分
3. IF 驾驶员累计扣分达到6分，系统 SHALL 生成警告通知
4. IF 驾驶员累计扣分达到12分，系统 SHALL 生成严重警告并建议暂停该驾驶员出车
5. WHEN 驾驶员扣分周期结束时，系统 SHALL 自动清零已到期扣分并保留历史记录

---

### R6 - 油耗异常智能预警引擎

**User Story:** AS 车队管理员, I want 油耗异常自动检测和预警, so that 及时发现车辆故障或油料异常消耗

#### Acceptance Criteria

1. WHEN 车辆完成加油记录录入，系统 SHALL 自动计算该车辆的百公里油耗
2. WHILE 车辆油耗数据累积超过30天，系统 SHALL 建立该车辆的正常油耗基准区间
3. IF 某次加油周期油耗偏离基准区间超过20%，系统 SHALL 生成油耗异常预警
4. IF 连续3个加油周期油耗持续偏高，系统 SHALL 建议安排车辆检修检查
5. WHEN 车队管理员查看油耗异常详情，系统 SHALL 展示异常油耗与基准值的对比图表

---

### R7 - 维修保养费用同比/环比分析

**User Story:** AS 财务人员, I want 维修保养费用的同比和环比分析, so that 掌握费用趋势、优化维修预算

#### Acceptance Criteria

1. WHEN 财务人员选择分析时间段，系统 SHALL 按车辆和维修类型展示维修保养费用汇总
2. WHEN 财务人员启同比分析，系统 SHALL 计算当前期间费用与去年同期费用的变化率
3. WHEN 财务人员启用环比分析，系统 SHALL 计算当前期间费用与上一期间费用的变化率
4. IF 某项费用同比或环比增幅超过预设阈值，系统 SHALL 高亮标记并提示异常
5. WHEN 财务人员导出分析报告，系统 SHALL 生成包含图表和数据的PDF或Excel格式报告

---

### R8 - 车辆全生命周期运营成本分析

**User Story:** AS 财务人员, I want 车辆全生命周期成本分析, so that 精确评估车辆资产效益、指导采购决策

#### Acceptance Criteria

1. WHEN 财务人员查看车辆TCO看板，系统 SHALL 展示每辆车的累计总成本，包括购置成本、燃油费、维修保养费、保险费、年检费
2. WHEN 财务人员选择成本构成维度，系统 SHALL 以饼图展示各成本项的占比
3. WHEN 财务人员对比多辆车成本，系统 SHALL 并排展示各车辆的TCO及每公里成本
4. WHILE 车辆处于运营状态，系统 SHALL 按月度自动汇总该车辆的各项费用
5. IF 某车辆累计维修成本超过购置成本的一定比例，系统 SHALL 生成车辆更换建议

---

### R9 - 移动端运营管理入口

**User Story:** AS 车队管理员, I want 移动端快捷操作入口, so that 随时随地进行车辆运营管理

#### Acceptance Criteria

1. WHEN 车队管理员登录移动端，系统 SHALL 展示运营概览仪表盘，包含车辆在线数、待处理工单、近期提醒
2. WHEN 车队管理员使用移动端，系统 SHALL 提供加油记录录入、维修工单创建、年检预约等核心操作功能
3. WHEN 驾驶员登录移动端，驾驶员 SHALL 查看个人扣分记录、归属车辆的维修保养计划和个人任务
4. WHEN 移动端收到新提醒，系统 SHALL 通过移动端推送通知用户
5. WHILE 移动端处于弱网环境，系统 SHALL 缓存最近浏览数据并支持离线查看

---

### R10 - 智能提醒推送

**User Story:** AS 车队管理员, I want 多场景智能提醒推送, so that 不遗漏关键运营节点

#### Acceptance Criteria

1. WHEN 系统检测到待提醒事件，系统 SHALL 根据事件类型和紧急程度自动匹配通知渠道（站内消息、移动推送）
2. WHEN 车队管理员查看消息中心，系统 SHALL 按未读优先和时间倒序展示所有提醒消息
3. WHEN 车队管理员处理某条提醒，系统 SHALL 记录处理人和处理时间并自动关闭对应提醒
4. IF 提醒超过24小时未被处理，系统 SHALL 升级提醒级别并通知上一级管理者
5. WHILE 系统处于正常运行状态，系统 SHALL 在每日早晨汇总当天到期的所有提醒并统一推送

---

### R11 - 组织架构与数据权限隔离

**User Story:** AS 系统管理员, I want 按组织架构隔离数据, so that 不同层级用户只能查看权限范围内的数据

#### Acceptance Criteria

1. WHEN 系统管理员管理组织架构，系统 SHALL 支持总公司-分公司-车队三级树形结构的创建和维护
2. WHEN 用户被分配至某组织节点，系统 SHALL 限定该用户仅能查看本节点及下级节点的数据
3. WHEN 上级组织用户查看数据，系统 SHALL 自动聚合下级组织所有数据
4. IF 用户尝试访问无权限的组织数据，系统 SHALL 拒绝请求并返回权限不足提示
5. WHEN 车队管理员在权限范围内操作，系统 SHALL 记录所有数据变更的操作人和所属组织信息
